# Projektstruktur

## Teil 1: Projektvorstellung & Bedarfsanalyse  
Analyse der aktuellen Unternehmenssituation und Definition der Anforderungen für die digitale Transformation.

## Teil 2: ERM (Entity-Relationship-Modell) & operative Datenbank  
Modellierung der Geschäftsprozesse in einem ER-Modell sowie Aufbau einer operativen Datenbankstruktur.

## Teil 3: Data Warehouse / Datenlager (Data Mart Konzept)  
Entwicklung einer analytischen Datenstruktur zur Unterstützung von Reporting und Entscheidungsfindung.

## Teil 4: Data Quality  
Analyse und Sicherstellung der Datenqualität hinsichtlich Vollständigkeit, Konsistenz und Genauigkeit.

# Projektkontext

Der Kunde ist das Unternehmen **„Immobilienvermittlung“** aus Köln, das seit den 1970er Jahren tätig ist und 16 feste Mitarbeiter beschäftigt. Das Unternehmen operiert bundesweit, hat jedoch keine internationalen Kunden. Der Ansprechpartner ist Herr Frank Teske.

Im Rahmen einer Modernisierung soll der Betrieb künftig auf IT-basierte Prozesse umgestellt werden. Derzeit existiert jedoch kein integriertes IT-System.
